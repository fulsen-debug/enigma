1) What AI Guardian Does

AI Guardian is a Solana behavioral risk scanner for trader decision support.

  • Analyzes kill-switch risk and holder concentration before entry
  • Tracks behavior changes during live monitoring
  • Provides structured decision context: favorable, caution, high risk

2) What AI Guardian Does Not Do

  • Does not guarantee profits
  • Does not replace your execution discipline and stop rules
  • Does not remove the need to validate data on external market tools

3) Daily Workflow

  1. Connect wallet
  2. Run a random scan or paste one token for manual scan
  3. Review verdict, Market Regime panel (Volatility Index + ADX), and risk details before considering entry
  4. Use alerts to detect important status or risk changes while monitoring

Scanner access requires a KOBX balance. Tiers: 500,000 KOBX = 2 scans/day, 3,000,000+ KOBX = 5 scans/day.

4) Verdict Interpretation

  • FAVORABLE: cleaner profile, still verify externally
  • CAUTION: mixed signals, reduce size or skip until clearer
  • HIGH_RISK: block by default unless strict override policy exists

5) Alerting and Monitoring

Enable browser notifications and sound buzz for fast reaction while monitoring a token.

  • Use scanner alerts for status flips and fast risk changes.
  • Use the paper trade monitor to review executed BUY and SELL actions.
  • Use system messages for engine pauses, blocked entries, and runtime notes.

6) Random Scan Mode

Random scan surfaces one candidate token at a time. It is probabilistic and can include high-risk tokens.

  • Always run manual due diligence on every random candidate.
  • Do not auto-buy from random scan output without strict policy checks.

7) Team Operating Policy (Recommended)

  • No entry when verdict is HIGH_RISK
  • Require dual confirmation for CAUTION entries
  • Record post-trade outcomes for process improvement
  • Review missed alerts and false positives weekly

8) Trading Console (Paper Agent)

  • Set one Agent Token Mint for the paper agent loop. This is single-token mode.
  • Scanner and agent are separate. Scanner does quick token analysis; agent manages one selected token.
  • Use the top controls only: Paper Budget, Test Amount Per Trade, Check Every (sec), Run Test, and Stop.
  • Use Run Test, then validate results in the buy/sell visual activity panel.
  • Engine now uses a regime policy matrix (ADX + Volatility + liquidity/spread + holder risk) to auto-select breakout/trend/mean-revert or no-trade mode.
  • Entry logic is multi-trigger: pullback entry, momentum continuation, and timeout fallback (reduced size) to avoid waiting forever.
  • Position sizing is dynamic: risk-budget and stop-distance based, then reduced when volatility/spread rises.
  • Exit watcher supports fast multi-sweep checks for TP/SL/trailing/hostile-regime exits.
  • PnL logs now show expected vs realized outcomes and include execution drag (fees/slippage/latency).
  • Decision engine is deterministic and numeric (no LLM dependency in trade decisions).
  • Trade Activity table now prints BUY/SELL logs only; INFO/engine status stays in monitor/system messages.
  • Layered same-token entries are supported up to Max Open Positions with Open Cooldown control.
  • Check Every (sec) controls reaction speed: lower values react faster to TP/SL/trailing events, higher values reduce API load.
  • Engine guardrails auto-adjust invalid combinations so SL and Trailing Stop stay below TP.
  • Each paper run auto-stops after 8 minutes to enforce session caps.
  • Live execution is not part of this first release surface.

9) Holder Behavior Coverage Limits

The holder panel is intentionally transparent about analysis depth.

  • Holder concentration uses getTokenLargestAccounts top-20 largest token accounts through Helius-compatible RPC with standard RPC fallback.
  • Wallet buy/sell activity prefers Helius parsed wallet history when HELIUS_API_KEY or HELIUS_API_KEYS is configured; otherwise it falls back to RPC transaction parsing.
  • Cluster links and bundle/coordinated-risk verdicts remain AIG inference built on top of verified provider inputs.
  • Scanner share cards and X share text include both verified facts and inferred conclusions.

11) Export and Sharing Workflow

  • Use Download PNG in the scanner card to export a branded report image.
  • Use Share on X to open a prefilled post with scanner verdict, concentration, and forensic summary.
  • Use the scanner card itself as the primary operator report. It includes verified facts, AIG inference, holder behavior, and external links.

12) Integrating Scanner Reports Into Another System

Typical usage pattern:

  1. Authenticate the wallet session with /api/auth/nonce and /api/auth/verify.
  2. Request a scan with /api/signal.
  3. Optionally request deeper holder behavior with /api/token/holders.
  4. Render the returned verdict and risk fields in your own dashboard or trade page.
POST /api/signal
{
  "mint": "48iJcUv9jsiZ7cCisyVFLPFLMoNBKg3L43bRvktXpump"
}

The response includes the full scanner report: token, market, kill-switch, market regime, trader plan, rug risk, forensics, links, and holder behavior summary.
